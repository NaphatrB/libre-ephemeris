# Libre Ephemeris — Project Plan

A clean-room MIT-licensed rewrite of the Swiss Ephemeris astronomical computation library.
Rust implementation with a C ABI (`le_` prefix). No affiliation with Astrodienst AG.

## License & IP

- **License**: MIT
- **Clean-room**: All implementation derived from public-domain sources (NASA JPL
  technical reports, IAU standards, VSOP2013/ELP-MPP02 published data, Hipparcos
  catalog, published scientific papers). No reference to the AGPL'd Swiss Ephemeris
  source code. Regression tests compare against the original compiled binary output,
  never its source.

## Ephemeris Engines

Two engines, no proprietary `.se1` format:

| Engine | Source | Data dep | Precision |
|--------|--------|----------|-----------|
| **JPL reader** | Published NASA DE format specs (DE430/431/441) | ~2 GB DE file | ~0.001" |
| **Analytical** | VSOP2013 (planets) + ELP-MPP02 (Moon) | Zero files | ~0.1" |

**Build order**: JPL reader first (simpler, establishes truth), analytical second.

## API Scope

C ABI with `le_` prefix, same numeric constant values as the original.
Rust internal API uses idiomatic types; C ABI generated via `cbindgen`.

**Tier 1 (MVP)** — all must ship:
- Planetary positions (indices 0–17: Sun through Earth, incl. Chiron)
- JPL DE file reader (430/431/441)
- Analytical engine (VSOP2013 + ELP-MPP02)
- Julian day / calendar conversions
- Delta T (all models: SM 1984, SM 1997, EM 2006, SMH 2016, Schoch)
- Ayanamsa (all 47 predefined modes)
- Coordinate transforms (all standard flags: equatorial, ecliptic, of-date, J2000, ICRS, sidereal, topocentric)
- Precession (all 11 IAU models)
- Nutation (IAU 2000A + 2000B)
- Aberration, gravitational deflection, frame bias
- Topocentric (observer position via WGS84 ellipsoid)
- Houses (all 20+ systems)
- Fixed stars (full Hipparcos catalog, build-time download & embed)
- Error handling (int return + char* buffer, matching original pattern)
- Thread safety (TLS default context + explicit handle API)
- Coordinate transforms (`le_cotrans`)
- Version, close, set_ephe_path, set_jpl_file

**Tier 2 (post-MVP)**:
- Eclipse calculations
- Rise/set/transit
- Refraction / azimuth-altitude
- Planetary phenomena (magnitude, phase)

**Tier 3 (post-MVP)**:
- Heliacal phenomena
- Event search (aspects, ingresses, transits)

## Build System & Platform

- **Build**: Cargo (Rust toolchain)
- **C ABI header**: generated by `cbindgen` via `build.rs`
- **Platforms (MVP)**: Linux x86_64, macOS x86_64 + ARM, Windows x86_64
- **Versioning**: semver from 0.1.0

## Data Pipeline

- **Hipparcos catalog**: `build.rs` downloads from CDS, parses, embeds as compact
  binary blob via `include_bytes!`
- **Leap seconds**: `build.rs` downloads from IERS, embeds as static array
- **VSOP2013 coefficients**: downloaded from IMCCE at build time, parsed into
  embedded coefficient tables
- **ELP-MPP02 coefficients**: same approach as VSOP2013

## Testing Strategy

1. **Unit tests** — internal math functions
2. **Black-box regression** — feed 1000+ dates/flags to both `original_binary` and
   our library, compare 6-vectors. Tolerance: 0.001" for JPL engine, 1" for analytical.
3. **Property-based** — continuity (positions change smoothly), conservation
   (transform round-trips recover original), speed consistency
4. **Benchmarks** — detect regressions on hot paths (Chebyshev eval, trig series)
